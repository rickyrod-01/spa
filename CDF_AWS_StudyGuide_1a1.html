Question 10

Question 10: When replicating Data Sharing Agreements(DSA's) in DSA Replication Prime NiFi Flow, which of the following is true? (select 3)

  1. Both the Prime and Failover clusters execute a cron job under svc_pmt to store the DSA's in their local HDFS system
  2. Both the Prime and Backup clusters execute a cron job under svc_policy to store the DSA's in their local HDFS system
  3. A node cron job from the Prime cluster to Failover cluster via Policy Management Tool (PMT) is created
  4. A node cron job from the Backup clusters via PMT is created
  5. NiFi is used on the Failover cluster to overwrite its local PMT DSA and ARC directories
  6. NiFi is used on the Backup cluster to duplicate its local PMT DSA and ARC directories

Question 10 Answer1, 3 and 5 are the correect answers (p. 85 Data Ingest Guide)

Question 11

Question 11: The Cron job associated with a DSA Replication Prime NiFi Flow is ... (select all that are true)

  1. Required on the PMT node of each cluster.
  2. Will run every weekday at 9 and 3 pm.
  3. Will run every weekday at 9 pm and 3 am
  4. Has 2 parts 1) Creating the Ticket Granting Ticket (TGT) and Executing the HDFS command to add the DSA's
  5. Has 3 parts; Granting the Master Ticktet (MT), executing the Kafka flow for each DSA, and Creating a backup DSA Flow
  6. A Kerberos Keytab is required so that the "kinit" as the service payment user will NOT be prompted for a password.

Question 11 AnswersAnswers are 1,2,4 and 6. See page 85 under 'Cron job" in the Data Ingest Guide for an explanation
